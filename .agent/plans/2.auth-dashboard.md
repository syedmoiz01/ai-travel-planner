# 2. Auth + User Dashboard

✅ **Simple** — standard email/password JWT auth, no external accounts needed.

## Scope

- Email/password registration + login (JWT bearer tokens)
- SQLite via SQLAlchemy (file DB, upgradeable to Postgres later via `DATABASE_URL`)
- Saved trips: persist a generated itinerary against a user, list/delete on a dashboard
- Google/Apple login deferred — those need OAuth app credentials from an external
  console (Google Cloud / Apple Developer), out of scope until requested

## Tasks

1. Backend: `db.py` (engine/session), `db_models.py` (`User`, `SavedTrip`), `auth.py`
   (password hashing, JWT issue/verify, `get_current_user` dependency)
   - **Validation**: register a user via curl, log in, hit `/api/auth/me` with the token
2. Backend: `/api/trips` CRUD (create on save, list mine, delete)
   - **Validation**: save an itinerary via curl with a valid token, list it back
3. Frontend: login/register pages, auth context (token in localStorage), header shows
   login state
   - **Validation**: register + log in through the browser, header updates
4. Frontend: dashboard page listing saved trips; "Save trip" button on `/plan`
   - **Validation**: generate an itinerary, save it, see it on `/dashboard`
