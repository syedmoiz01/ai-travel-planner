# 1. MVP Scaffold — AI Travel Planner

⚠️ **Medium** — scoped down from a 22-deliverable production spec to a single-pass buildable slice.

## Why this scope

The original spec (landing page + AI itinerary + destination info + route optimization +
cost calculator + hotel booking APIs + deal finder + chat assistant + auth + admin dashboard +
K8s/CI/CD) is a multi-month production build. This plan builds the smallest end-to-end slice
that proves the concept, with clean seams to extend later:

- Real AI itinerary generation (actual LLM call, structured output)
- Mocked hotels/weather/attractions data, shaped like the real APIs they'll be swapped for
- One working page-to-page flow: search → generated itinerary

Everything else in the original spec (auth, booking partner APIs, maps routing, admin dashboard,
deployment infra) is deferred to later plans (see Roadmap below).

## Stack

- Frontend: Next.js 14 (App Router) + TypeScript + Tailwind + shadcn/ui
- Backend: FastAPI (Python) + Pydantic structured outputs
- LLM: OpenAI or OpenRouter (configurable via env var), no LangChain
- Data: mocked hotels/weather/attractions modules shaped like target real APIs (Google Places,
  OpenWeather, Amadeus) so swapping later is a data-source change, not a schema change

## Tasks

### 1. Backend scaffold
- FastAPI app with `/api/itinerary` (POST) and `/api/destinations/{name}` (GET, mocked data)
- Pydantic models: `TripRequest`, `Itinerary`, `DayPlan`, `MockDestinationInfo`
- `.env.example` with `OPENAI_API_KEY`, `OPENROUTER_API_KEY`, `LLM_MODEL`
- **Validation**: `curl -X POST localhost:8000/api/itinerary -d '{...}'` returns a valid
  structured itinerary JSON matching the Pydantic schema; `/api/destinations/tokyo` returns mock data.

### 2. Frontend scaffold
- Landing page: hero, search form (destination, dates, travelers, budget, style chips), submit button
- Itinerary results page: day-by-day cards (morning/afternoon/evening/night), mocked hotel/weather cards
- Dark/light mode toggle
- **Validation**: manual browser walkthrough — fill search form, submit, see a real
  AI-generated itinerary render without errors; toggle dark mode.

### 3. Wire frontend → backend
- Next.js route calls FastAPI `/api/itinerary`, renders response
- Loading and error states
- **Validation**: full flow in browser with backend running locally, network tab shows
  successful request/response.
